# AI产品岗位能力地图

## 文档目的

这份文档不是原样堆叠 JD，而是把你收集到的大量岗位要求做一次去重、归类和取舍，回答三个问题：

1. 真正高频出现的核心能力是什么
2. 哪些能力应该先学，哪些可以后置
3. 哪些条目其实是噪音，不该继续占用你的注意力

## 结论先行

如果你的目标是 **AI 产品经理 / AI Agent 产品经理 / AI 工具产品实习**，这些 JD 的真正交集并不多，核心只有 8 个模块：

1. 大模型与 Agent 的基本技术理解
2. Prompt 设计与效果调优
3. Agent 工作流编排与工具调用
4. API 调用与快速原型能力
5. 用户研究、竞品分析、需求拆解
6. PRD / 原型 / 交互方案设计
7. 数据分析、评测、Badcase 迭代
8. 跨团队协作与项目推进

换句话说，你现在不缺“更多要学的东西”，而是缺一个更干净的优先级系统。

## 去重后的能力框架

### P0：必须优先建立的核心能力

#### 1. LLM / Agent 基础认知

你至少要能说清：

- LLM 能做什么、不能做什么
- Agent 和普通问答式 AI 的区别
- ReAct、Plan-and-Execute、多 Agent 这些词分别在解决什么问题
- Prompt、RAG、Fine-tuning、Tool Use 分别应该在什么场景下用

对应到岗位里，这类要求高频出现在：

- AI 产品经理
- AI Agent 产品经理
- 大模型平台/应用产品实习
- AI 工具产品实习

#### 2. Prompt Engineering

这不是“会写几句提示词”，而是能系统设计：

- Role
- Context
- Task
- Constraint
- Example

同时你还要知道：经典的 CoT、few-shot、zero-shot CoT 是重要概念，但对新的 reasoning 模型不能机械照搬，应该先用简单直接的 prompt，再决定是否补示例。

#### 3. Agent 工作流编排

这是你从“会用 AI”走向“会做 AI 产品”的分水岭。

你需要至少亲手做过：

- 一个 Coze Bot / Workflow
- 一个 Dify Workflow 或 Chatflow
- 一个可解释的 Agent 执行链路

不是为了会某个平台，而是为了真正理解：

- 哪些步骤适合交给 Agent 自主决策
- 哪些步骤必须用显式流程固定下来
- 什么地方要加条件分支、重试、日志和人工兜底

#### 4. API 调用与快速原型

很多岗位把它写成 Python / JavaScript / vibe coding / API 调用能力，本质上是在看你能不能把一个 AI 能力快速接到真实产品里。

最低要求不是“做工程师”，而是：

- 知道模型调用的基本结构
- 理解 `model`、上下文、输出控制、错误处理、速率限制、成本
- 能配合研发看懂简单调用代码
- 最好自己接通过 1 到 2 个 demo

#### 5. 用户研究与需求分析

大多数 JD 都会换一种说法重复它：

- 用户洞察
- 用户需求分析
- 市场与竞品研究
- B 端场景理解
- 产品意识与逻辑思维

这类能力的核心不是“会写分析报告”，而是能判断：

1. 用户到底在解决什么问题
2. AI 适不适合解决这个问题
3. 应该把 AI 放在流程的哪个位置
4. 如果 AI 失败，用户还能不能继续完成任务

#### 6. 产品方案与交互表达

高频对应词：

- PRD
- 原型设计
- 交互设计
- 用户流程
- 多模态交互

你不一定要成为设计师，但要能把方案表达清楚，让研发和设计知道要做什么、为什么这样做、验收标准是什么。

#### 7. 数据分析、评测、Badcase

这是 AI 产品区别于普通产品的重要部分。

你需要开始有这套意识：

- 不只看 DAU、留存、转化
- 还要看任务完成率、对话轮次、误答率、用户反馈、Token 成本
- 任何体验问题最终都要进入 Badcase 记录、分析、验证闭环

#### 8. 协作与推进

这是几乎所有产品岗都会看，但很多人最晚才补。

实际要练的是：

- 会前准备
- 需求评审
- 向上汇报
- 会后纪要
- 任务 owner、deadline、风险同步

这部分很不“酷”，但真正决定你能不能把 AI 产品做成。

### P1：建立核心能力之后补齐

这些能力重要，但不应该先于上面的 P0：

1. 多模态交互设计
2. Benchmark 设计与模型效果评估
3. Fine-tuning / RAG / Embedding 的边界判断
4. B 端产品经验方法论
5. LangChain / Langflow / 更复杂的 Agent 框架
6. SQL / 埋点 / 可视化分析能力
7. 市场分析、行业研究、商业模式分析

### P2：加分项，不要提前焦虑

这些能力当然有价值，但如果 P0 还没打稳，就不该占据你当前主线：

1. 云平台经验（AWS / Azure / GCP / 阿里云）
2. 开源经验 / 深度开发经验
3. 内容创作 / 视频剪辑 / 审美表达
4. 英语优势
5. 海外平台使用经验
6. 证书类加分项

## 应该主动丢掉的噪音

你给的清单里混入了不少 **不属于当前主线** 的内容。

如果你的目标是 AI 产品方向，这些内容目前不该继续挤占你的注意力：

1. HRBP / 招聘 / HR 全面能力
2. 游戏海外用户研究这条完整职业线
3. 与目标岗位无关的泛化软素质堆叠

原因不是它们没价值，而是它们会制造“我什么都要学”的错觉，直接破坏推进感。

## 建议学习顺序

### 第一阶段：先建立最小闭环

目标：从“知道概念”进入“做出东西”。

建议顺序：

1. Prompt 基础
2. LLM / Agent 基础概念
3. Coze 或 Dify 二选一先跑通
4. API 调用一个最小 demo
5. 写一份简短 PRD

阶段产物建议：

- 一个 Bot
- 一个工作流
- 一个 API demo
- 一份 1 到 2 页的需求说明

### 第二阶段：把产品能力接进技术能力

目标：不只是会搭流程，而是能解释为什么这么设计。

建议顺序：

1. 用户研究与竞品分析
2. 交互方案与失败兜底
3. Badcase 表设计
4. 核心指标与评测维度

阶段产物建议：

- 一份竞品拆解
- 一份用户任务流程图
- 一张 Badcase 表
- 一套效果评估指标

### 第三阶段：形成作品化表达

目标：把学习资产转成可展示的项目。

建议组合：

1. 一个 AI Agent 产品 case study
2. 一个工作流设计说明
3. 一份带指标和 Badcase 的迭代复盘

## 你当前最该沉淀的 5 个知识库主题

基于这批 JD，当前最值得进入知识库主干的不是“职位大全”，而是下面 5 个主题：

1. Prompt Engineering 与 reasoning 模型使用原则
2. Coze / Dify / Langflow 的定位差异与学习顺序
3. AI 产品经理的核心指标与 Badcase 机制
4. AI Agent 产品从需求到工作流的设计方法
5. AI 产品岗的能力优先级与作品集路线

## 一句话版本

你的问题不是“要学的太少”，而是“信息过多但主线不清”。

当前正确方向是：

**先把 Prompt + Agent Workflow + API Demo + 用户研究 + 评测闭环 这 5 件事做成，再谈扩展。**

## 执行要求

这份能力地图默认服务于 **高强度短周期冲刺**，不是松散学习。

如果你的目标是短时间内拿出能投递的材料，建议直接配合：

- `00-项目说明/AI产品一周冲刺计划.md`

用 7 天把核心能力压缩成：

1. 至少 3 个 demo
2. 至少 2 份 case study
3. 至少 1 套评测 / Badcase 材料
