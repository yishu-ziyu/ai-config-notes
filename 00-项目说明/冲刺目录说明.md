# 冲刺目录说明

## 目录用途

从现在开始，这个仓库不只记录知识，也承接 7 天冲刺的实际产物。

目录分工如下：

### `01-学习卡片/`

沉淀可复用的知识卡片，不写当天流水账。

适合放：

- Prompt 模板卡
- 模型能力边界卡
- 平台差异卡
- 指标定义卡

### `02-demo/`

放每天做出来的 demo 说明、实验记录和执行结果。

适合放：

- demo 设计说明
- Prompt 对比实验
- API 调用实验
- 工作流节点说明

### `03-作品集材料/`

放最终要用于投递和展示的内容。

适合放：

- 项目总览
- case study
- 竞品分析摘要
- 方案设计说明

### `04-评测与Badcase/`

放评测、指标、badcase 和优化记录。

适合放：

- Badcase 表
- 测试样例集
- 评测维度说明
- 优化前后对比

### `99-收件箱/`

临时内容先落这里，之后必须整理，不允许长期堆积。

## 使用原则

1. 学习资料不能直接等于作品集材料，必须二次加工。
2. demo 记录和 case study 要分开，不要混写。
3. Badcase 必须单独归档，否则后续无法复盘。
4. 每天至少有一个文件进入 `02-demo/` 或 `03-作品集材料/`。

## 命名约定

推荐从现在开始逐步采用下面的命名格式：

```text
D{day}_{topic}_{artifact-type}_v{version}.md
```

例如：

```text
D01_prompt-compare_demo-brief_v1.md
D03_dify-workflow_node-map_v1.md
D06_eval-badcase_eval-report_v1.md
```

约定如下：

1. `D01-D07` 固定两位数，保证排序稳定。
2. 文件名优先使用英文 slug，不用空格，不用中文标点。
3. `artifact-type` 尽量从固定集合里选：`card`、`demo-brief`、`prompt-spec`、`node-map`、`user-flow`、`run-log`、`result-log`、`eval-set`、`eval-report`、`badcase-log`、`before-after`、`case-study`、`prd`。
4. 最终作品集成品不必保留 `Dxx` 前缀，可以使用 `CS01`、`PRD01`、`A01` 这类更适合投递的编号。
