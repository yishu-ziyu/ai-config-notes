# AI产品一周冲刺计划

## 目标

这一版不是“学习计划”，而是 **7 天冲刺式作品集生产计划**。

目标只有 3 个：

1. 在 7 天内补齐 AI 产品岗最关键的能力最小集
2. 每天必须产出一个可展示 demo 或半成品
3. 在 7 天结束时，手上至少有一套可以投递的作品材料

## 冲刺原则

### 1. 每天都必须有产物

每天结束前必须留下至少一个可见产物：

- demo
- PRD
- 流程图
- 竞品分析
- Badcase 表
- case study

### 2. 只学会立刻进入作品集的内容

任何学习动作都必须回答：

**它今天会进入哪个 demo 或哪份作品材料？**

如果回答不上来，就先不学。

### 3. 先做能跑的，再做更漂亮的

优先级如下：

1. 能跑通
2. 能解释
3. 能复盘
4. 能包装

## 7 天后的最低交付标准

### Demo

至少 3 个：

1. 一个 Prompt / 对话优化类 demo
2. 一个 Agent 工作流 demo
3. 一个 API 接入或多模型实验 demo

### 作品集材料

至少 5 份：

1. 一份 AI 产品项目总览页
2. 一份 Agent 产品 case study
3. 一份竞品分析或用户需求分析
4. 一份评测 / Badcase 迭代材料
5. 一份 PRD 或流程设计文档

### 过程资产

至少 3 类：

1. Prompt 模板
2. 指标与评测表
3. Badcase 记录表

## 推荐目录

建议从现在开始在仓库里逐步使用这套结构：

```text
00-项目说明/
01-学习卡片/
02-demo/
03-作品集材料/
04-评测与Badcase/
99-收件箱/
```

## 每日节奏

建议每天按 4 段推进：

1. 上午：补理论，只看今天要用到的部分
2. 中午前：做最小 demo
3. 下午：补成方案文档或 case study
4. 晚上：写复盘，记录 badcase 和明日任务

## Day 1：Prompt 与模型能力边界

### 学习目标

建立对 Prompt、reasoning、few-shot、zero-shot、模型边界的基本判断。

### 必做事项

1. 读 Prompt Engineering 与 Reasoning Best Practices
2. 总结 1 页 Prompt 模板卡
3. 用同一个任务测试 2 到 3 种 Prompt 写法

### 当日 demo

做一个 `Prompt 对比 demo`：

- 同一个任务
- 至少 3 种 Prompt 版本
- 对比输出质量、稳定性、成本

### 当日作品集材料

1. `Prompt 模板卡`
2. `模型能力边界笔记`
3. `Prompt 优化前后对比页`

### 当日复盘问题

1. 什么样的 Prompt 改动最有效
2. 什么问题不是靠 Prompt 就能解决

## Day 2：Coze Bot 与单 Agent 设计

### 学习目标

把 Prompt 设计转成可运行的 Bot。

### 必做事项

1. 读 Coze Agent 文档
2. 做一个单 Agent Bot
3. 明确这个 Bot 的用户、场景、输入、输出、限制

### 当日 demo

做一个 `Coze 单 Agent demo`，比如：

- 简历优化助手
- 面试复盘助手
- 会议摘要助手
- AI 学习路径助手

### 当日作品集材料

1. `Bot 定位说明`
2. `核心 Prompt 说明`
3. `用户使用流程图`

### 当日复盘问题

1. 哪些任务适合单 Agent
2. 哪些需求开始需要流程编排

## Day 3：Dify Workflow / Chatflow

### 学习目标

把“会写 Bot”升级成“会设计结构化 AI 流程”。

### 必做事项

1. 读 Dify Workflow & Chatflow、Dify Agent
2. 做一个最小 workflow
3. 设计分支、条件、输入变量和输出结构

### 当日 demo

做一个 `Dify 工作流 demo`，例如：

- 用户输入目标岗位 -> 输出能力差距分析 -> 输出学习建议
- 用户上传 JD -> 提取技能 -> 给出优先级建议

### 当日作品集材料

1. `工作流节点图`
2. `为什么不用纯聊天而要用 workflow`
3. `关键节点说明`

### 当日复盘问题

1. 哪一步最适合显式流程控制
2. 哪一步最容易出 badcase

## Day 4：API 调用与快速原型

### 学习目标

补齐“能和研发对话、能自己做最小原型”的能力。

### 必做事项

1. 选一个模型平台，接通最小 API demo
2. 记录核心参数、错误处理、速率限制和成本
3. 做一个最简单的前端或脚本式原型

### 当日 demo

做一个 `API 原型 demo`：

- 输入一段岗位描述
- 输出能力标签、优先级和学习建议

### 当日作品集材料

1. `API 接入说明`
2. `参数说明卡`
3. `成本与风险说明`

### 当日复盘问题

1. 产品经理真正需要理解哪些工程参数
2. 哪些问题必须和研发一起定义

## Day 5：用户研究、竞品与需求拆解

### 学习目标

避免只会搭 demo，不会定义问题。

### 必做事项

1. 选择一个 AI 产品方向做竞品分析
2. 写目标用户、核心痛点、任务路径
3. 明确 AI 在这个流程里的位置和失败兜底方式

### 当日 demo

今天不追求新技术 demo，重点做 `需求定义 demo`：

- 一页竞品矩阵
- 一页用户任务流程
- 一页需求优先级

### 当日作品集材料

1. `竞品分析页`
2. `用户流程图`
3. `需求拆解说明`

### 当日复盘问题

1. 哪个需求是真需求
2. 哪个需求其实不需要 AI

## Day 6：评测、指标与 Badcase

### 学习目标

从“做出来”升级到“知道怎么持续迭代”。

### 必做事项

1. 为已有 demo 设计评测维度
2. 建一张 Badcase 表
3. 用真实案例记录至少 10 个 badcase

### 当日 demo

做一个 `评测与 Badcase demo`：

- 选一个前几天做的 demo
- 跑一批测试样例
- 记录成功率、失败类型、修正策略

### 当日作品集材料

1. `评测指标说明`
2. `Badcase 分类表`
3. `优化前后对比`

### 当日复盘问题

1. 最常见失败是什么
2. 应该改 Prompt、改流程还是改产品边界

## Day 7：打包成作品集

### 学习目标

把前 6 天的分散产物整理成可以投递、可以讲述的材料。

### 必做事项

1. 选出 2 个最强 demo
2. 各写一份 case study
3. 整理总览页，把所有产物连起来

### 当日 demo

今天的 demo 不是新功能，而是 `作品集成品`：

- 总览页
- 2 个 case study
- 附录：Badcase / 指标 / Prompt / 流程图

### 当日作品集材料

1. `项目总览`
2. `案例一：从需求到 Agent 工作流`
3. `案例二：从 Prompt 到评测迭代`

### 当日复盘问题

1. 哪个案例最能打动面试官
2. 哪块还缺关键证据

## 每天晚上必须写的 5 条记录

1. 今天做了什么
2. 今天产出了什么文件
3. 今天最重要的一个 badcase
4. 明天最重要的唯一目标
5. 今天哪部分已经可以进作品集

## 一周结束后的判断标准

这一周是否成功，不看你看了多少资料，只看这 4 件事：

1. 你是否做出了至少 3 个 demo
2. 你是否写出了至少 2 份能讲清楚的 case study
3. 你是否建立了自己的 Badcase 与评测材料
4. 你是否能把这些内容讲成一个完整的 AI 产品故事

## 一句话版本

这不是“7 天学完 AI 产品”，而是：

**7 天把 AI 产品岗最关键的能力压缩成一组可展示、可复盘、可投递的作品。**
