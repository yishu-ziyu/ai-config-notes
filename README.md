# AI 知识库

这个仓库不再定位为“泛泛记录 AI 内容”，而是一个面向未来复用的 **AI 配置、接入、工作流、踩坑复盘** 知识库。

## 这个项目的意义

这个仓库应该回答的不是“我最近看了什么 AI 内容”，而是：

1. 我以后还会重复做什么 AI 相关动作？
2. 哪些配置、接入、工具链、工作流值得沉淀成可复用资产？
3. 下一次我再做同类事情时，怎样 5 分钟内找到可执行方案？

如果一条内容不能帮助未来的你更快完成任务，它就不应该成为这个仓库的核心内容。

## 收录原则

优先收录这三类内容：

1. 配置卡：某个工具/服务怎么接入，关键参数是什么，风险点是什么。
2. 工作流卡：某类任务如何从输入走到输出，步骤、判断条件、产物是什么。
3. 复盘卡：一次实际踩坑中，问题、原因、解法、以后如何避免。

不再把它当作：

1. AI 新闻收藏夹
2. 零散灵感堆放区
3. 没有结论的随手抄录本

## 推荐目录

建议后续逐步整理为下面这套结构，不需要一次性搬完：

```text
00-项目说明/
01-配置接入/
02-工作流/
03-案例复盘/
99-收件箱/
```

其中：

- `01-配置接入/`：邮箱、平台、CLI、模型服务、账号权限等接入说明
- `02-工作流/`：例如“如何把一个 AI 工具接入日常工作”
- `03-案例复盘/`：真实问题、真实过程、真实结论
- `99-收件箱/`：临时记录，之后必须二次整理

## 最小维护方式

每新增一条笔记，至少补齐这 4 项：

1. 这条笔记解决什么问题
2. 适用场景是什么
3. 关键步骤或判断点是什么
4. 下次我会如何找到并复用它

## 当前内容的合理起点

从现有内容看，这个仓库最自然的起点不是“大而全的 AI 知识库”，而是：

**个人 AI 工具与办公自动化配置手册**

这个方向更窄，但更容易形成持续推进感，也更容易产出真实可复用内容。
