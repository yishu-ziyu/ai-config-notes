# D02 Coze Single Agent Demo Brief

## Demo 名称

AI 产品求职学习助手

## 这个 demo 想证明什么

不是证明我会搭平台。

而是证明我知道：

- 什么问题适合单 Agent
- 单 Agent 的边界该怎么定
- 输出怎么设计得更像产品，而不是一段泛泛聊天

## 用户是谁

准备投 AI 产品经理 / AI Agent 产品经理岗位的人。

## 用户会输入什么

1. 目标岗位
2. 当前背景
3. 目前最薄弱的地方
4. 可投入时间

## Agent 要输出什么

1. 当前差距判断
2. 最重要的 3 个学习重点
3. 一周行动建议
4. 不该优先投入的内容

## 为什么用单 Agent 就够了

因为这个任务当前只需要：

- 理解输入
- 做出判断
- 给出一次结构化输出

它还没有复杂到必须拆分工作流。

## 这次 demo 的成功标准

1. 输出不发散
2. 输出像产品建议，不像闲聊
3. 能直接进入知识库或作品集
4. 用户看完知道下一步该做什么

## 这次 demo 的风险

1. 建议太空
2. 输出太长
3. 没有优先级
4. 假装什么都懂

## 面试时怎么讲

这不是一个技术很重的 Agent。

它更像一个产品判断 demo。

我想展示的不是“多智能”，而是“边界清楚、任务清楚、输出清楚”。
