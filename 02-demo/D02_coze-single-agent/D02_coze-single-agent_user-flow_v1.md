# D02 Coze Single Agent User Flow v1

## 用户路径

### Step 1

用户给出目标。

例如：

- 我想投 AI 产品经理
- 我现在不会写 Prompt
- 我只有 7 天

### Step 2

Agent 判断信息够不够。

如果不够，它只追问最关键的问题。

例如：

- 你现在有没有实际做过 demo？
- 你每天大概能投入多久？

### Step 3

Agent 给出差距判断。

不是泛泛分析。

而是告诉用户：

- 你现在最缺什么
- 什么先做
- 什么别急着做

### Step 4

Agent 输出一周行动建议。

输出应该让用户立刻能开始，而不是看完更焦虑。

## 这条路径为什么成立

因为用户真正需要的不是“更多信息”。

而是：

**更清楚的下一步。**

## 这条路径最容易坏在哪里

1. Agent 一上来讲太多
2. Agent 不追问关键缺口
3. 输出没有优先级
4. 输出像鸡汤，不像方案
