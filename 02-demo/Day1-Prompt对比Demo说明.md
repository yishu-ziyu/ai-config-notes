# Day1 Prompt 对比 Demo

这次实验其实只想回答一个问题：

同一个模型，为什么有的 Prompt 一看就不对，有的 Prompt 却会明显更稳？

## 任务

给一份很长、很乱、很多重复的岗位能力清单。

让模型提炼出真正重要的能力。

## 我做了什么

我写了 3 个版本。

### A

很短，很泛。

本质上就是一句：

“帮我总结一下这个岗位需要什么能力。”

### B

开始把任务讲清楚。

有角色，有上下文，有任务，也有限制。

### C

不再只是“让它总结”。

而是明确告诉它：

- 这是知识库任务
- 不是泛泛总结
- 要有优先级
- 要有固定输出
- 还要写清楚什么不该优先投入

## 结果

A 最像“会说很多，但不够准”。

B 已经明显更像一个可用结果了。

C 最像真实工作里能拿去用的东西。

## 结论

好 Prompt 和差 Prompt 的区别，不在于谁写得更复杂。

而在于谁更知道自己要什么。

差 Prompt 常见的问题是：

- 任务太泛
- 边界不清
- 输出没有标准

好 Prompt 的核心是：

- 任务清楚
- 输入清楚
- 输出清楚
- 限制清楚

## 这次实验最重要的一句话

Prompt 不是为了显得专业。

Prompt 是为了让模型少犯错。
