# Day1 Prompt 对比 Demo 说明

> 目标：用同一任务、同一输入、多个 Prompt 版本，展示 Prompt 迭代对结果的影响。

## 1. Demo 目标

- 展示内容：`用同一份岗位能力原始材料，对比 3 种 Prompt 写法在“去重、优先级判断、可执行性”上的表现差异`
- 想验证的问题：`结构化 Prompt 是否比泛化总结 Prompt 更适合 AI 产品经理的知识沉淀任务`
- 面向作品集的结论：`当任务目标是知识库沉淀而不是泛化总结时，明确输出结构、优先级字段和排除项的 Prompt 明显更优`

## 2. Demo 结构

1. 同一个任务
2. 同一份输入
3. 3 个 Prompt 版本
4. 对比输出质量、稳定性、成本
5. 得出最终推荐版本

## 3. 任务与输入

- 任务名称：`岗位能力清单去重与优先级提炼`
- 输入样例：`见 ../99-收件箱/inbox_2026-04-18_day1_jd_raw_input.md`
- 输入规模：`10 条代表性岗位能力摘录`
- 是否固定输入：`是`
- 额外约束：`同模型、同任务、同输入；只比较 Prompt 写法带来的差异`

## 4. Prompt 版本对比

| 版本 | 模型 | Prompt 特征 | 输出特征 | 稳定性 | 成本 | 结论 |
| --- | --- | --- | --- | --- | --- | --- |
| A | MiniMax-M2.7-highspeed | 泛化总结、低约束 | 给出 Top10 和学习路径，但模块偏散、结果偏“教学模式” | 单次观察偏低 | 1677 tokens | 可读，但不适合直接入库 |
| B | MiniMax-M2.7-highspeed | Role + Context + Task + Constraint | 成功收束到 8 个模块，结构更清楚 | 单次观察中 | 1805 tokens | 已接近可用，但仍偏长且被截断 |
| C | MiniMax-M2.7-highspeed | 明确输出结构 + 排除项 + 字数限制 | 结论最集中，优先级最清楚，最贴合知识库任务 | 单次观察中高 | 1838 tokens | 当前最佳版本 |

## 5. 展示重点

- 为什么选这几个版本：`A 代表自然总结，B 代表标准结构化 Prompt，C 代表面向知识库沉淀的强约束 Prompt`
- 哪个版本最稳定：`当前单次观察是 C，但还需要重复实验确认`
- 哪个版本最省成本：`A，输入约束最少、token 最低`
- 哪个版本最适合上线：`C，因为它最接近“产品可用输出”而不是“学习型长答”`

## 6. Demo 结论

- 推荐版本：`C`
- 推荐理由：`它最好地兼顾了去重、优先级、可执行性和知识库适配性，在相同模型与输入下输出最聚焦`
- 不推荐版本的问题：`A 过于泛化；B 虽然更好，但仍然容易展开过长并触发截断；三者都受到 max_tokens 限制`
- 下一步优化：`增加多轮重复测试；进一步缩短输出预算；单独加入“不得展开教学式解释”的约束；尝试关闭或规避隐藏思考开销`
