为什么它差
  • 它知道要总结，但不知道要怎么总结。
  • 它给了很多内容，却没有真正收束成判断。
  • 它更像一篇“会写很多”的答案，而不是一个能直接拿去用的结果。
Good Prompt

它知道你要的不是“总结”，而是“判断”。

Prompt 把任务、边界、输出结构和排除项都讲清楚了，所以结果更聚焦。

Prompt
你现在扮演岗位分析师。

任务：
根据下面的岗位描述，提炼 5 个最核心能力，并按 P0 / P1 / P2 排优先级。

输出要求：
1. 用中文
2. 输出表格，字段为：能力 / 为什么重要 / 优先级
3. 不要照抄原文
4. 不要超过 200 字

材料：
[岗位描述]
真实输出片段
# 核心能力模块

| 模块 | 包含能力 | 为什么重要 | 优先级 |
| Prompt 与 Agent | Prompt 调优、Agent 工作流设计、Badcase 反馈机制 | 面试必问、实操必考 | P0 |

| 技术理解力 | LLM/NLP、RAG/Fine-tuning、API、Coze/Dify | 决定你能否与研发平等对话 | P1 |
| 协作与推进 | 沟通、项目推进、自驱力 | 决定方案能不能落地 | P1 |
为什么它好
  • 它把“你要做什么”说清楚了。
  • 它把“结果要长什么样”说清楚了。
  • 它把“什么不能做”也说清楚了。
最重要的不是 Prompt 写得多复杂。
最重要的是，Prompt 有没有让模型更少误解你。
