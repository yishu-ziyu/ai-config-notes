# Inbox Triage Rules

## 用途

防止仓库重新退化成日记仓库。

## 进入收件箱的内容

1. 原始链接
2. 临时灵感
3. 还没验证的想法
4. 零散截图与素材
5. 还不能进入作品集的半成品

## 升级规则

### 升级到 `01-学习卡片/`

当内容已经提炼出稳定结论、可复用方法或明确边界判断时。

### 升级到 `02-demo/`

当内容已经变成可执行实验、流程或原型，并有输入输出记录时。

### 升级到 `03-作品集材料/`

只有同时满足以下 4 条才允许：

1. 有明确用户问题
2. 有可见方案或 demo
3. 有评测或 badcase 证据
4. 有你的决策与反思

### 升级到 `04-评测与Badcase/`

当内容已经涉及测试样例、失败案例、指标定义或优化前后对比时。
