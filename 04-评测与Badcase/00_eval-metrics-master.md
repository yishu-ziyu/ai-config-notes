# Eval Metrics Master

## 用途

统一整轮冲刺的评测口径，避免每个 demo 各写一套指标，后续无法合并成作品集。

## 通用指标

| 指标 | 定义 | 适用场景 | 目标值 | 备注 |
| --- | --- | --- | --- | --- |
| 任务完成率 | 用户目标是否完成 | Prompt / Agent / Workflow |  |  |
| 输出准确率 | 输出是否正确 | 分类 / 抽取 / 问答 |  |  |
| 格式正确率 | 输出是否符合预期结构 | JSON / 表格 / 规范化输出 |  |  |
| 稳定性 | 多次运行结果是否一致 | Prompt / Workflow |  |  |
| 平均时延 | 完成一次任务的时间 | Agent / API |  |  |
| Token 成本 | 单次或单批处理成本 | Prompt / API |  |  |
| 人工介入率 | 需要人工兜底的比例 | Agent / Workflow |  |  |

## 使用规则

1. 每个 demo 至少选 2 个主指标。
2. 结果必须能回填到作品集 case study。
3. 不能只写感受，必须有定义和样本来源。
