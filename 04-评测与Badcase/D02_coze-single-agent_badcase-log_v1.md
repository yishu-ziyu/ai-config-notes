# D02 Coze Single Agent Badcase Log v1

## 先记 4 类最可能出现的问题

### 1. 建议太空

表现：
说了很多正确的话，但用户不知道今天该做什么。

处理：
强制输出“本周 3 个重点”和“一周行动建议”。

### 2. 话太多

表现：
Agent 一次说太长，像文章，不像助手。

处理：
限制输出结构和长度。

### 3. 假装很懂用户

表现：
用户没说清楚背景，Agent 却直接给了很具体的判断。

处理：
信息不够时，先追问 1 到 2 个关键问题。

### 4. 边界不清

表现：
用户问到不该由这个 Agent 负责的事，它也硬答。

处理：
明确写出“它负责什么，不负责什么”。

## 这份 badcase log 的作用

Day 2 不是为了证明 Agent 很聪明。

而是为了证明我知道：

单 Agent 最容易在哪些地方坏掉。
