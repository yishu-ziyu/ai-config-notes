# Day1 Prompt 模板卡

> 目标：把同一个任务拆成可对比、可复用、可复盘的 Prompt 模板。

## 1. 任务定义

- 任务名称：`岗位能力清单去重与优先级提炼`
- 使用场景：`面对超长、重复、混杂的 AI 产品岗位能力清单时，快速提炼真正高频的能力模块，并用于生成学习计划`
- 目标用户：`准备 AI 产品经理 / AI Agent 产品经理岗位的求职者`
- 期望结果：`输出去重后的核心能力模块、P0/P1/P2 优先级和一周最小学习闭环`
- 成功标准：`能去掉重复表述；能把能力收束到 5-8 个模块；能给出可执行优先级；结果适合直接进入知识库`

## 2. 任务输入

- 输入类型：`长文本`
- 输入内容示例：`见 ../99-收件箱/inbox_2026-04-18_day1_jd_raw_input.md`
- 必要字段：`岗位名称、能力项、优先级描述、相关平台或技术关键词`
- 可选字段：`额外岗位备注、加分项、非目标方向噪音项`
- 限制条件：`输入存在大量近义重复、不同岗位混杂、部分信息与当前主线无关`

## 3. 输出要求

- 输出格式：`结论摘要 + 表格 + 优先级 + 一周建议`
- 输出长度：`800 字左右，必要时允许稍长`
- 必须包含：`核心能力模块、优先级判断、可执行建议、去重后的表达`
- 禁止包含：`大段照抄原始材料、泛泛鼓励、与当前目标无关的职业线`
- 质量标准：`准确 / 可执行 / 可解释 / 便于进入知识库`

## 4. Prompt 版本

### 版本 A

```text
请阅读下面的岗位能力清单，提炼出最重要的能力，并按优先级排序。

要求：
1. 用中文回答
2. 尽量简洁
3. 给出学习建议

材料：
[粘贴原始材料]
```

- 设计目的：`测试“泛化总结型 Prompt”在真实 JD 提炼任务上的表现`
- 适用模型：`MiniMax-M2.7-highspeed`

### 版本 B

```text
你是一名 AI 产品经理求职教练。

Context:
我正在根据多份 AI 产品/Agent 产品岗位要求，提炼真正高频的能力要求。输入材料存在大量重复、近义表达和不同岗位混杂。

Task:
请你完成 4 件事：
1. 去重并合并相近能力
2. 提炼 6-8 个核心能力模块
3. 按 P0 / P1 / P2 分类
4. 给出一周内最值得优先建立的最小闭环

Constraint:
- 用中文
- 不要照抄原文
- 不要把软素质堆成列表
- 结果要能直接用于学习计划
- 输出格式固定为：
  A. 核心能力模块表
  B. P0/P1/P2 列表
  C. 一周最小闭环

材料：
[粘贴原始材料]
```

- 设计目的：`测试 Role + Context + Task + Constraint 结构对输出质量的提升`
- 适用模型：`MiniMax-M2.7-highspeed`

### 版本 C

```text
你现在扮演“AI 产品岗位能力分析师”。

请基于下面的原始材料，输出一个“可直接进入知识库”的结论。

你的目标不是总结，而是帮助用户完成 Day1 Prompt 实验，因此结果必须体现：
- 去重能力
- 结构化能力
- 优先级判断
- 可执行性

请严格按以下结构输出：

# 结论摘要
用 3 条要点说清这批岗位真正高频的能力主线。

# 核心能力模块
用表格输出，字段为：模块 / 包含哪些能力 / 为什么重要 / 优先级。

# 一周冲刺建议
只保留最重要的 5 项任务，按先后顺序输出，每项都要说明“为什么这一项现在必须做”。

# 不应优先投入的内容
列出 3 项当前不该优先深挖的方向，并说明原因。

约束：
- 用中文
- 不要抄原文句子
- 不要超过 800 字
- 风格务实、像给求职中的 AI 产品经理做判断

材料：
[粘贴原始材料]
```

- 设计目的：`测试“明确结构 + 严格输出格式 + 排除项”是否能显著提升知识库可用性`
- 适用模型：`MiniMax-M2.7-highspeed`

## 5. 对比维度

| 维度 | A | B | C | 备注 |
| --- | --- | --- | --- | --- |
| 模型 | MiniMax-M2.7-highspeed | MiniMax-M2.7-highspeed | MiniMax-M2.7-highspeed | 同模型对比 |
| 输入约束 | 低 | 中 | 高 | 约束越明确，结果越可控 |
| 输出结构 | 弱 | 中 | 强 | C 最接近知识库格式 |
| 稳定性 | 单次观察偏低 | 单次观察中 | 单次观察中高 | 未做多轮重复实验 |
| 成本 | 1677 tokens | 1805 tokens | 1838 tokens | 以 input+output token 粗略比较 |
| 响应速度 | 中 | 中 | 中 | 体感差异不大 |
| 可解释性 | 中 | 高 | 高 | B/C 明确体现去重逻辑与判断依据 |
| 适配场景 | 泛化总结 | 学习计划生成 | 知识库沉淀 / 作品集材料 | C 最贴合当前任务 |

## 6. 结论草稿

- 最优版本：`C`
- 选择原因：`在相同模型与相同输入下，C 最好地压缩了能力模块数量，并且同时给出了结论摘要、优先级和“不应优先投入”的判断，更适合直接进入知识库或作品集`
- 不足点：`3 个版本都触发了 max_tokens 截断，说明当前 API 的隐藏思考或输出预算会影响结果完整性`
- 下一轮优化方向：`进一步压缩输出字数；增加强制字段限制；做 2-3 次重复运行验证稳定性；单独评估“是否照抄原文”的问题`

## 7. 作品集表述

- 这张卡想证明什么：`我不是只会写提示词，而是能围绕真实任务设计 Prompt 版本、比较结果、做优先级判断，并把实验结论沉淀为产品资产`
- 可以放进作品集的一句话：`我用同一份 AI 产品岗位原始材料对比了 3 种 Prompt 设计方式，验证了“结构化约束”比“泛化总结”更适合知识库沉淀和产品化输出。`
