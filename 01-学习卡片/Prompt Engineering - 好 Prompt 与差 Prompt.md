# Prompt Engineering：好 Prompt 与差 Prompt

很多人一开始学 Prompt，会误以为重点是“写得高级”。

其实不是。

Prompt Engineering 的重点只有一句话：

**让模型更准确地理解你的任务。**

## 为什么要写 Prompt

因为同一个模型，输入写法一变，结果就会变。

有时候差别不是 10 分和 8 分。

而是“能不能用”的差别。

## 什么是差 Prompt

差 Prompt 最大的问题不是短。

而是含糊。

比如：

```text
帮我总结一下这个岗位需要什么能力。
```

这句话的问题在于：

- 任务不清楚
- 输出不清楚
- 优先级不清楚
- 成功标准不清楚

模型只能自己猜。

一旦它开始猜，结果就会飘。

## 什么是好 Prompt

好 Prompt 不一定长。

但它一定会把关键的东西讲清楚。

比如：

```text
你现在扮演 AI 产品岗位分析师。

任务：
根据下面的岗位描述，提炼 5 个最核心能力，并按 P0 / P1 / P2 排优先级。

输出要求：
1. 用中文
2. 输出表格，字段为：能力 / 为什么重要 / 优先级
3. 不要照抄原文
4. 不要超过 200 字

材料：
[岗位描述]
```

它更好的原因不是“更长”。

而是它把下面几件事说清楚了：

1. 你是谁
2. 你要做什么
3. 结果要长什么样
4. 什么不能做

## 我现在最认可的 Prompt 骨架

如果你以后自己写 Prompt，可以先从这个骨架开始：

```text
你现在扮演：[角色]

任务：[要完成什么]

输入：[会给你什么材料]

输出要求：
1. [格式]
2. [长度]
3. [必须包含什么]
4. [不要包含什么]
```

## 一个最重要的判断标准

Prompt 写得好不好，不看它漂不漂亮。

只看一件事：

**它有没有让模型更少误解你。**

## 可以怎么记笔记

以后每个 Prompt 案例都可以按这个最小结构记：

### 任务

这条 Prompt 想解决什么问题。

### 差 Prompt

写法是什么。

### 好 Prompt

写法是什么。

### 差异

为什么一个好，一个差。

### 我学到的东西

这次之后，我以后写 Prompt 会注意什么。

## 最后一句

公司不是为了让你写“漂亮 Prompt”。

公司是希望你用 Prompt，把任务做对。
