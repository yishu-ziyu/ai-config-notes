# Day 2：Coze 单 Agent 卡

Day 2 不追求复杂。

目标只有一个：

做出一个真正像产品，而不是像聊天记录的单 Agent。

## 这次做什么

这次我选的主题是：

**AI 产品求职学习助手**

它解决的问题很具体：

用户知道自己想投 AI 产品岗。
但不知道该先学什么，也不知道怎么把目标拆成一周内能执行的动作。

## 为什么这个题目适合单 Agent

因为它现在还不需要复杂编排。

它只需要把下面几件事做好：

1. 读懂用户目标
2. 判断用户现在的状态
3. 输出一个清楚、能执行的建议

这类任务的重点不是“多工具协作”。

而是：

**一次对话里，把问题理解对，把建议说清楚。**

## 单 Agent 最适合的任务

单 Agent 适合这些场景：

- 一个明确目标
- 一次核心输出
- 不依赖很多外部工具
- 重点在判断和表达

比如：

- 学习路径助手
- 简历优化助手
- 面试复盘助手
- 会议摘要助手

## 单 Agent 不适合的任务

如果出现下面这些情况，就要开始考虑 workflow：

- 需要多步决策
- 需要条件分支
- 需要多工具调用
- 需要中间状态保存
- 需要失败后重试或降级

## 这次这个 Agent 的边界

它负责：

- 理解用户目标岗位
- 提炼当前能力差距
- 给出一周学习重点
- 帮用户把任务压缩成可执行动作

它不负责：

- 直接判断用户一定能不能拿到 offer
- 生成虚假的经历
- 输出和目标岗位无关的长篇建议
- 替用户做最终决定

## Day 2 最重要的判断

一个好的单 Agent，不是“什么都能聊”。

而是：

**它很清楚自己该解决什么，不该解决什么。**
