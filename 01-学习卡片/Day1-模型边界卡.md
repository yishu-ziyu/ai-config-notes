# Day1 模型边界卡

这次实验让我确认了一件事。

Prompt 写得不好，模型就会开始“自作聪明”。

它不是不会写。
它只是根本不知道你到底要什么。

## 这次最明显的 3 个边界

### 1. 任务一模糊，模型就会开始发散

如果你只说“帮我总结一下”，模型大概率会写成一篇看起来很努力的长文。

问题不是它不聪明。
问题是它没有被你限制住。

### 2. 好 Prompt 不只是提升质量，也是在控制成本

这次 3 个版本都被截断了。

这说明一件很现实的事：

不是你说了“简洁”，模型就一定会简洁。

Prompt 还要负责控制输出预算。

### 3. 单次结果好，不等于这个 Prompt 稳

这次 C 看起来最好。

但“看起来最好”只是第一步。

真正有用的 Prompt，要经得起重复跑。

## 对我真正有用的结论

Prompt 不是写得漂亮。

Prompt 是让模型别误解你。

一个好 Prompt，至少要把下面 4 件事说清楚：

1. 你要它做什么
2. 你给它什么材料
3. 你想要什么样的输出
4. 什么不允许它做
